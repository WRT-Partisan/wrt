前言

标准模板库 STL（Standard Template Library），是 C++ 标准库的一部分，不需要单独安装，只需要 #include 头文件。

C++ 对模板（Template）支持得很好，STL 就是借助模板把常用的数据结构及其算法都实现了一遍，并且做到了数据结构和算法的分离。

STL六大部件

  • 容器（Containers）
  • 分配器（Allocators）
  • 算法（Algorithm）
  • 迭代器（Iterators）
  • 适配器（Adapters）
  • 仿函数（Functors）

要真正提高 C++ 编程效率，需要将 STL 六大部件结合使用，才能大放异彩。

vector

可变长度数组。

构造

一维数组

基本方法：vector<类型> 数组名(长度,[初值])（括号内的内容可以不写，默认长度为 000）。

构造数组，并用 a,b,c,…a,b,c,\ldotsa,b,c,… 初始化：vector<类型> 数组名{a,b,c,...};。

时间复杂度：O(n)O(n)O(n)。

二维数组

构造指定行数，不指定列数的二维数组：vector<类型> 数组名[行];（常用于邻接表存图）。

构造二维数组基本方法：vector< vector<类型> > 数组名;。

构造指定行、列、初值的二维数组：vector< vector<类型> > 数组名(行,vector<类型> (列,初值))。

sort

在 [first,last)[first,last)[first,last) 中进行排序。

comp 函数的返回值是一个 bool 值，当返回值为 true 时不改变元素顺序，反之则需要调换元素（可换可不换时要交换）。

lower_bound/upper_bound

二分查找。

在已升序排序的元素中，应用二分查找检索指定元素，返回对应元素迭代器位置。找不到则返回尾迭代器。

lower_bound(_ForwardIterator __first, _ForwardIterator __last,const _Tp& __val, _Compare __comp)

upper_bound(_ForwardIterator __first, _ForwardIterator __last,const _Tp& __val, _Compare __comp)

lower_bound()：寻找 ≥x\ge x≥x 的第一个元素的位置。

upper_bound()：寻找 >x>x>x 的第一个元素的位置。

返回的是迭代器，减去头迭代器即可转成下标索引。

比较器默认为 less<类型>（从小到大），若序列从大到小为 greater<类型>。

自定义比较

自定义 comp 函数。

bool comp(类型 前一个位置元素,类型 后一个位置元素);。

comp 函数的返回值是一个 bool 值，当返回值为 true 时往左找，反之向右找。

max/min

返回最大值/最小值的数值。

template< class T > const T& max( const T& a, const T& b );。

template <class T> const T& min (const T& a, const T& b);。

两个元素

max(第一个元素,第二个元素)。

min(第一个元素,第二个元素)。

多个元素

使用列表构造语法传入一个列表，这样就能一次性给多个元素找最值。

max({第一个元素,第二个元素,第三个元素,...})。

min({第一个元素,第二个元素,第三个元素,...})。

unique

消除数组的重复相邻元素，数组长度不变，但是有效数据缩短，返回的是有效数据位置的结尾迭代器。

ForwardIt unique( ForwardIt first, ForwardIt last );。

单独使用 unique 并不能达成去重效果，因为它只消除相邻的重复元素。但是如果序列有序，那么它就能去重了。

去重 vector 时序列尾部会产生一些无效数据，为了删掉这些无效数据，我们需要结合 erase。

数组.erase(unique(数组.begin(),数组.end()),数组.end())。
