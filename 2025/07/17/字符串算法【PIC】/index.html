前缀函数计算

概念

真前缀/真后缀：不包含本身的前缀/后缀。

border：一个字符串相等的真前缀与真后缀称为它的 border。如 s=s=s=abcdab，ab 是 sss 的 border。

前缀函数：对于 sss 的任意位置 xxx，定义 π(x)\pi(x)π(x) 为 S1→xS_{1\to x}S1→x​（前缀串）的最大 border。

例：

s :a b a c a b a b a c d a b a

pi:0 0 1 0 1 2 3 2 3 4 0 1 2 3

暴力

注意到 π(i)≤π(i−1)+1\pi(i)\le\pi(i-1)+1π(i)≤π(i−1)+1（每增加一个字符，最好的情况是上一个串的 border 再成功匹配一次）。

优化 2（穷举方式优化）

穷举 iii →\to→ 穷举长度 len∈[1,π(i−1)+1]len\in[1,\pi(i-1)+1]len∈[1,π(i−1)+1]（从大到小，找到停止）→\to→ 判断串相等。

匹配成功：nnn 次，“爬升”总数：≤n\le n≤n 格（每个点最多爬一格），匹配失败：小于等于爬升总数，即 ≤n\le n≤n。

时间复杂度：O(n2)O(n^2)O(n2)。

优化 3（穷举方式优化）

注意到由于每次成功匹配一定会用到 s[1,i−1]s[1,i-1]s[1,i−1] 的 border，如果要找 s[i−1]s[i-1]s[i−1] 的 π(i)\pi(i)π(i)，那么只要考虑 s[1,i−1]s[1,i-1]s[1,i−1] 的所有 border 即可，这样判断只需要一个字符即可，如果成功即结束，如果失败换下一个 border。

判断串相等时间复杂度 O(1)O(1)O(1)，总时间复杂度 O(n)O(n)O(n)。

在 sss 中找 www 出现的情况（即出现多少次和出现在哪里）。

扩展 KMP（Z 函数）

概念

对于字符串 sss 的每一个位置 xxx，找最长长度 lenlenlen，使得 s1→len=sx→x+len−1s_{1\to len}=s_{x\to x+len-1}s1→len​=sx→x+len−1​。

计算方法

暴力

枚举位置 iii，暴力匹配。时间复杂度：O(n2)O(n^2)O(n2)。

优化

l,rl,rl,r 为当前最长的前缀串的左端点和右端点。

贪心：rrr 越大，越可能包含后续的 iii，越可能少匹配，时间复杂度：O(n)O(n)O(n)。

  1. 注意到 lll 和 rrr 不断增大，因此它们的更新是 O(n)O(n)O(n) 的。
  2. 发现匹配的次数与 rrr 的增长是一一对应的，因此字符匹配的次数是 O(n)O(n)O(n)。
  3. 字符匹配失败的总次数明显是 O(n)O(n)O(n)。

马拉车（Manacher）

概念

计算序列中每一个中心位置能扩展到最大回文长度。

奇长回文：以项为中心。

偶长回文：以项间为中心。

计算以项为中心的最大回文半径

暴力

穷举中心，暴力扩展。时间复杂度：O(n2)O(n^2)O(n2)。

优化

如果中心 xxx 处于某个回文中（中心为 midmidmid，左边界为 lll，右边界为 rrr）。

对于 xxx 在回文中的镜像点 x′x'x′，有：

radix≥radix′=radil+r−xradi_x\geq radi_{x'}=radi_{l+r-x}radix​≥radix′​=radil+r−x​。

radix=min⁡(r−x+1,radil+r−x)radi_x=\min(r-x+1,radi_{l+r-x})radix​=min(r−x+1,radil+r−x​)。

[l,r][l,r][l,r]：rrr 最大的回文区间。

时间复杂度分析：

  1. 匹配失败最多 nnn 次（每个中心一次）。
  2. 匹配成功必须在大于 rrr 的情况，且每次匹配成功下一次 rrr 会加 111，因此匹配成功的总次数等于 rrr 的总移动次数，因此小于等于 nnn。

所以时间复杂度为 O(n)O(n)O(n)。

计算以项间为中心的最大回文半径

在任意两个字符间隔之间加上一个字符（如 #），再在两边都加一个边界符（如 ()），跑 Manacher 求出新串不包含中心的半径，等价于回文直径。
