B - Reverse Proxy

题目大意

问题陈述

有 NNN 个编号为 1,2,…,N1,2,\dots,N1,2,…,N 的盒子。初始时，所有盒子都是空的。

QQQ 个球会依次出现。

高桥将按照 X=(X1,X2,…,XQ)X=(X_1,X_2,\dots,X_Q)X=(X1​,X2​,…,XQ​) 的顺序把球放入盒子中。

具体来说，他会对 iii 这个球进行以下处理：

  • 如果 Xi≥1X_i \ge 1Xi​≥1 ：将此球放入 XiX_iXi​ 盒子。
  • 如果 Xi=0X_i = 0Xi​=0 ：将此球放入装球数量最少的盒子中。

找出每个球被放入哪个盒子。

限制因素

  • 所有输入值均为整数。
  • 1≤N≤1001 \le N \le 1001≤N≤100
  • 1≤Q≤1001 \le Q \le 1001≤Q≤100
  • 0≤Xi≤N0 \le X_i \le N0≤Xi​≤N

思路

第一种操作直接加一，输出 XiX_iXi​；第二种操作暴力寻找最小值，加一并输出最小值下标。

C - Rotatable Array

题目大意

问题陈述

有一个长度为 NNN 的整数序列 AAA ，初始值为 Ai=iA_i = iAi​=i 。请处理以下类型的共计 QQQ 次查询：

  • 类型 111 ：将 ApA_pAp​ 更改为 xxx 。
  • 类型 222 ：输出 ApA_pAp​ 。
  • 类型 333 ：重复 "将 AAA 的第一个元素移至末尾 "的操作 kkk 次。形式上，用 (A2,A3,…,AN,A1)(A_2,A_3,\dots,A_N,A_1)(A2​,A3​,…,AN​,A1​) 替换 AAA kkk 次。

限制因素

  • 所有输入值均为整数。
  • 1≤N≤1061 \le N \le 10^61≤N≤106
  • 1≤Q≤3×1051 \le Q \le 3 \times 10^51≤Q≤3×105
  • 所有查询均满足以下约束条件：
    • 1≤p≤N1 \le p \le N1≤p≤N
    • 1≤x≤1061 \le x \le 10^61≤x≤106
    • 1≤k≤1091 \le k \le 10^91≤k≤109

思路

操作三相当于将起始下标向右移，用 ititit 记录偏移量，操作一和二下标加上偏移量即可。注意边界值和取余 nnn。

D - XOR Shortest Walk

题目大意

问题陈述

有一个有向图，图中有 NNN 个顶点和 MMM 条边，顶点的编号从 111 到 NNN ，边的编号从 111 到 MMM 。边 iii 是一条从顶点 AiA_iAi​ 到顶点 BiB_iBi​ 的有向边，权重为 WiW_iWi​ 。

求从顶点 111 到顶点 NNN 的有向边异或的权重的最小值。

限制因素

  • 2≤N≤10002 \leq N \leq 10002≤N≤1000
  • 0≤M≤10000 \leq M \leq 10000≤M≤1000
  • 1≤Ai,Bi≤N1 \leq A_i,B_i \leq N1≤Ai​,Bi​≤N
  • 0≤Wi<2100 \leq W_i < 2^{10}0≤Wi​<210
  • 所有输入值均为整数。

思路

一开始想BFS，设置一个极大值，遍历次数超过极大值遍历就停止，在提交七次最好一次 WA 两个点后发现玄学并不可靠。于是注意到 0≤Wi<2100 \leq W_i < 2^{10}0≤Wi​<210，是一个比较小的数字，于是搞一个标记数组，记录从 111 到每个点的每个异或值是否出现过，若没有出现过就向前走。

E - Battles in a Row

题目大意

问题陈述

高桥要玩一个游戏，他要按顺序与 NNN 个怪物战斗。最初，高桥的健康为 HHH ，魔力为 MMM 。

在与 iii 只怪物战斗时，他可以选择以下行动之一：

  • 不使用魔法战斗。只有当他的生命值至少为 AiA_iAi​ ，且生命值减少 AiA_iAi​ 且怪物被击败时，他才能选择这个行动。
  • 使用魔法战斗。只有当他的魔力至少为 BiB_iBi​ ，且魔力减少 BiB_iBi​ ，并且怪物被击败时才能选择。

当所有 NNN 个怪物都被打败或他无法采取任何行动时，游戏结束。在游戏结束前，他最多可以打败多少只怪物？

限制因素

  • 1≤N≤30001 \leq N \leq 30001≤N≤3000
  • 1≤H,M≤30001 \leq H,M \leq 30001≤H,M≤3000
  • 1≤Ai,Bi≤30001 \leq A_i,B_i \leq 30001≤Ai​,Bi​≤3000
  • 所有输入值均为整数。

思路

和上周模拟赛的第二题有点像，考虑 DP。

dp[i][j]dp[i][j]dp[i][j] 代表击败前 iii 个怪物后，消耗生命值为 jjj 时的最小魔力消耗。

初始化：将 dpdpdp 设置为极大值，dp[0][0]=0dp[0][0]=0dp[0][0]=0，即未攻击任何怪物时消耗为 000。
转移：

  1. 若生命值消耗 Ai+j≤HA_i+j\le HAi​+j≤H，那么 dp[i][j+Ai]=min⁡(dp[i][j+Ai],dp[i−1][j])dp[i][j+A_i]=\min(dp[i][j+A_i],dp[i-1][j])dp[i][j+Ai​]=min(dp[i][j+Ai​],dp[i−1][j])。
  2. 若魔法值消耗 dp[i−1][j]+Bi≤Mdp[i-1][j]+B_i\le Mdp[i−1][j]+Bi​≤M，那么 dp[i][j]=min⁡(dp[i][j],dp[i−1][j]+Bi)dp[i][j]=\min(dp[i][j],dp[i-1][j]+B_i)dp[i][j]=min(dp[i][j],dp[i−1][j]+Bi​)。

若在转移过后发现没有一种情况的魔法消耗小于 MMM，则结束循环，输出循环次数即可。
